História da Web

O Início

Joseph Carl Robnett Licklider cria a teoria sobre uma rede galática que permitia conexão de todas as pessoas. Após ser contratado pela DARPA graças a sua teoria, a DARPA começou a pesquisar e depois de um tempo desenvolveu a ARPANET, uma predecessora da Internet.

A ARPANET usava um Sistema de Comunicação em Pacotes que eram enviados um a um. E usavam nós, que eram pontos de intersecção de informações funcionando como "checkpoints" (eram parecidos com os roteadores que temos hoje em dia).

Primeira Conexão (e queda de conexão)

A primeira conexão aconteceu em 29/10/1969 e também a primeira queda de conexão, a primeira mensagem transmitida foi "lo" (seria "login" se não fosse pela "queda" de conexão)

Como chegamos até aqui?

Com o passar do tempo houve a criação de inúmeras sub-redes, para diversos fins, porém não havia nada muito amigável para o público geral, cada um fazia o seu "site" da sua forma. Vendo isso Tim Berners-Lee em 1989 decidiu usar hipertexto junto com o protocolo TCP/IP para criar a WWW

Lee também foi responsável por criar o primeiro navegador que se chamava WorldWideWeb e criou o primeiro site da Internet.

O que são Clients?

Client é o nome dado aos computadores que acessam e possibilitam ao usuário desfrutar dos recursos do servidor acessado.

Navegadores

Os navegadores são a forma pela qual o usuário é capaz de visualizar a página html, eles vão na internet e baixam a página e são responsáveis por otimizar o acesso a aquela página, e por interpretar os códigos, os estilos, e aplicar tudo isso no HTML.

Cada navegador tem o seu jeito de interpretar uma página web, dizemos isso, pois nem todos os navegadores vão dar suporte imediato para uma tag nova específica, e isso gera problemas com a compatibilidade, apesar de tudo, isso não é um problema que o torne o site inutilizável, já que a maioria dos sites utilizam tags básicas e que a maioria dos navegadores já oferecem suporte. Se um navegador vai dar suporte ou não, depende da versão que o usuário está usando que pode ou não incluir o suporte.

O que são Servers?

São máquinas que geralmente possuem alto poder de processamento e muito espaço de armazenamento, e servem para fornecer serviços para o usuário.

Tipos de Servidores:

  • De Arquivos: Armazenam arquivos para que os clients possam acessa-los posteriormente.

  • De Segurança (Firewall): Geralmente usado para impedir acessos externos.

  • De Streaming: Servidores feitos para guardar vídeos e por mandá-los para o usuário de maneira rápida e otimizada, para que o usuário possa ir consumindo o conteúdo.

  • De E-mail: Responsáveis por armazenar e-mails e gerenciá-los.

  • Web: Armazenam páginas web, mais especificamente os arquivos HTML, CSS e Javascript que são responsáveis por compôr a página web.

  • DNS: Servidor com uma lista de domínios devidamente relacionados com os endereços IP do servidor que está armazenando os arquivos do site.

  • Proxy: Responsável por mediar a comunicação entre o Client e a internet, muito usado em empresas para impedir que os funcionários acessem sites perigosos, dando mais segurança para a empresa.
    • Em alguns casos ele pode impedir que um site que não esteja na lista de bloqueio, de ser aberto para o usuário (ele bloqueia a resposta do site), então se algo do tipo ocorrer o primeiro passo é usar os dados móveis para checar se é só na rede com proxy ou se é um problema do site em si.

Um servidor não necessariamente precisa ser um servidor acessível apenas via internet, qualquer computador pode ser um servidor, você pode usar uma conexão local mesmo para acessar e mandar dados para o seu servidor.

Quando temos vários servidores em um local para servir um ou mais serviços aos usuários, chamamos este local de Data Center.

Hospedagem de Sites

  1. Registrar o domínio
    • Sempre procure registrar o domínio antes de qualquer coisa, se outro registrar o mesmo domínio que você quer usar, você terá que comprar o domínio dessa pessoa.
    • Sugestão de lugar para registrar o seu domínio:
      Registro.br

  2. Encontrar uma empresa de Hospedagem
    • Um mesmo servidor pode ser usado para mais de um servidor (Email e Web, por exemplo)
      • Isso pode deixar o site mais lento, pois o processamento do computador fica ocupado com os outros serviços. O ideal é ter uma máquina para cada serviço (um para armazenar e-mails e outro para guardar os arquivos do site).
    • Certificado SSL: Responsável por servir como a base para a criptografia dos sites que usam o protocolo HTTPS, assim mesmo que alguém consiga acessar os dados do site, ele terá descriptografar os dados para poder usa-los.

  3. Registrar o endereço DNS do servidor onde você planeja hospedar o site, no domínio.

  4. Hospedar o site no Servidor contratado usando o FTP
    • O servidor contratado fornecerá os dados para acessar o servidor (via FTP)
    • Recomendação de programa para facilitar: Filezilla

    • FTP - Conexão para enviar e receber arquivos, uma conexão direta com o servidor.
